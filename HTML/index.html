Etiquetas semanticas estructurales

Es una etiqueta contenedora que no tiene semántica
header-cabecera de un sitio web ó de una sección
main - Define la accion principal del documento, solo puede existir una etiqueta main por documento. EN ESTA PARTE ESTA EL CONTENIDO PRINCIPAL DEL DOCUMENTO
footer - pie de pagina de un sitio web o de una sección del sitio
nav - Representa una navegación: es decir un conjunto de enlaces que formen un menu de navegación ó una botonera
article - representa una sección autocontenido (que por si sola se explica) osea entiendo todo el contenido sin tener contexto de toda la interface del sitio. EJM: blog. la parte donde va todo el articulo del blog este seria por ejemplo un articulo.
aside- representa contenido complementario o secundario ejm: dentro de un article(lista-articulos relacionados) EJM: sitios donde hay que meter ZONAS DE PUBLICIDAD
section - Representa una seccion de contenido generico
address - Representa una dirección física
☝️

Etiquetas de bloque VS Etiquetas de linea

div - es la etiqueta de bloque por excelencia (sección de bloque generico, no tiene carga semantica) es una etiqueta que genera saltos de linea con sus etiquetas hermanas
span - es la etiqueta de linea por excelencia (contenedor generico en linea y no tiene carga semantica) solo ocupa el espacio necesario que tiene su contenido ☝️
